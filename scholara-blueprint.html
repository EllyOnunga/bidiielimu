🎨 White-Label Theming
Every school tenant can override the primary navy with their own brand color via the settings panel. Teal accent and Ghost White base remain constant to preserve readability and brand consistency across all 14 modules.
📐 Tailwind CSS Integration
All palette tokens are registered in tailwind.config.js as custom colors. This ensures your design system is enforced at the component level and stays consistent as the team scales.
🌙 Dark Mode Support
The Ghost White / Navy relationship inverts cleanly for dark mode. Teachers working late benefit from a dark sidebar with Ghost White content panels — reducing eye strain without redesigning the system.

Implementation Blueprint

Technology Stack

As recommended in the source document — proven, production-ready tools for each layer of the platform.

🐍 Backend Layer
  • Django 5 + DRF
    REST API, serializers, viewsets, signals
  • django-tenants
    Schema-per-tenant multi-tenancy
  • Django Channels
    WebSocket layer for real-time features
  • Celery + Redis
    Async task queue — SMS, emails, reports
  • SimpleJWT
    Stateless auth with refresh token rotation
  • scikit-learn
    Predictive grading ML models
⚡ Frontend Layer
  • Vite 6 + React 19
    SPA with lightning-fast HMR
  • Tailwind CSS v4
    Utility-first with custom palette tokens
  • TanStack Query v5
    Data fetching, caching, optimistic updates
  • Recharts + D3.js
    Analytics dashboards and data viz
  • Zustand
    Lightweight global state management
  • React Hook Form + Zod
    Forms and runtime validation
🗄️ Data Layer
  • PostgreSQL 16
    Primary DB — one schema per tenant
  • Redis 7
    Session cache, Celery broker, WS state
  • AWS S3 / Cloudflare R2
    Media, documents, report storage
  • Elasticsearch
    Fast full-text search across students
🚀 DevOps Layer
  • Docker + Docker Compose
    Consistent dev/prod environments
  • Nginx + Gunicorn
    Production reverse proxy & WSGI server
  • GitHub Actions CI/CD
    Automated tests, build, and deploy
  • Sentry
    Error tracking across all tenant apps
  • AWS / Hetzner Cloud
    Scalable Kenyan/EU hosting options

Development Roadmap

4-Phase Launch Plan

A realistic phased approach — ship fast, validate with real schools, then layer in intelligence and scale.

Phase 1 · Months 1–3 Foundation
Core Platform & First Tenant
  • Django project setup with django-tenants + PostgreSQL schema isolation
  • DRF API architecture — auth, tenants, base models
  • Vite + React shell — routing, Tailwind palette, layout system
  • Super-admin panel — tenant onboarding and billing dashboard
  • SIS (Student Information System) — profiles, admission, CBC strand support
  • RBAC — all 7 roles with granular permissions
  • Attendance management with SMS alerts
  • Docker containerization + CI/CD pipeline (GitHub Actions)
Phase 2 · Months 4–6 Academic Engine
Exams, Timetable & Finance
  • Automated timetable generator with constraint-solving algorithm
  • Full examination module — CATs, end-of-term, national exam simulation
  • Report card engine with grade auto-computation and PDF export
  • Fee management + M-Pesa STK Push + Africa's Talking SMS
  • Django Channels WebSocket — real-time grade and fee notifications
  • Parent portal PWA (offline-first with service workers)
  • Staff & HR module — contracts, leave, P9 generation
Phase 3 · Months 7–9 Intelligence Layer
AI, Analytics & LMS
  • AI Analytics Dashboard — Recharts + D3.js visualizations
  • Predictive Grading Engine (scikit-learn) — at-risk student flags
  • Attendance-Performance correlation analytics
  • LMS — assignments, resource library, auto-marking quizzes
  • Inventory & Asset Tracking module
  • Communication Hub — bulk SMS, notice board, event calendar
  • White-label theme engine — per-tenant CSS variable injection
Phase 4 · Months 10–12 Scale & AI+
Advanced AI, Mobile & Growth
  • AI Report Card Comment Generator (LLM integration)
  • React Native mobile app (iOS + Android) — student and parent facing
  • Stripe / PayPal international payment gateway for non-Kenya markets
  • Multi-language support (EN, SW, FR) for East/West African expansion
  • Elasticsearch full-text search across students, staff, documents
  • Discipline module with CBC co-curricular portfolio tracking
  • Tenant-level custom domain support (portal.greenwood.edu)
  • Public Scholara Marketplace — plugins by third-party developers
